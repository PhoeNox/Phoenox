---
name: Analyze vs Implement Distinction
description: When asked to analyze or find improvements, present findings only — do not execute changes without being asked
type: feedback
---

When asked to analyze, review, or find improvements, present the findings as a plan or list and stop there. Do not make code changes unless explicitly asked to implement them.

**Why:** The user corrected this after I analyzed layout issues on the Playlists page and immediately applied CSS changes, when they only wanted to see the findings first.

**How to apply:** Treat "find improvements", "review", "audit", "what should be changed" as requests for analysis output, not implementation. Only write code when the user says to apply, fix, implement, or do it.
